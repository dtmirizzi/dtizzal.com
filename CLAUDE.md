# dtizzal.com

Static site hosted on GitHub Pages. Vanilla HTML/CSS/JS — no build step for the pages themselves. CI builds the search index, sitemap, and llms.txt on push.

## Add a new blog post

```sh
python3 scripts/new-blog-post.py "Your Post Title"
```

Optional flags:
- `--description "Short SEO description"` (defaults to title)
- `--slug custom-slug` (defaults to slugified title)
- `--date "March 2026"` (defaults to current month)
- `--dry-run` to preview without writing

The script:
1. Picks the next post number (max existing + 1).
2. Creates `blog-posts/N-slug/slug.html` from `blog-posts/template.html` with `{{TITLE}}` and `{{DESCRIPTION}}` substituted.
3. Inserts a new `<li>` at the top of `blog.html`.

It does **not** touch `data/blog-index.json`, `sitemap.xml`, or `llms.txt`. Those are auto-generated by CI on push.

## After scaffolding — write the post, then ship

1. Open the new file at `blog-posts/N-slug/slug.html` and replace the placeholder `<h1>`, lede, sections, and footnotes with real content.
2. Preview locally:
   ```sh
   python3 -m http.server 8765
   open http://localhost:8765/blog-posts/N-slug/slug.html
   ```
3. Commit and push to `master`. GitHub Pages deploys in ~1 minute.
4. **CI auto-updates `data/blog-index.json`, `sitemap.xml`, and `llms.txt`** via `.github/workflows/build-blog-index.yml`. It runs on every push to master that touches `blog-posts/**/*.html`, top-level `*.html`, the resume, or any `scripts/build-*.js`. The workflow re-commits the regenerated files with message `chore: auto-update blog search index, sitemap, and llms.txt`. Don't edit those three files by hand — your edits will be overwritten on the next push.
5. After CI runs, `git pull` to sync the auto-update commit before doing more work.

## Verify llms.txt and sitemap after pushing

```sh
curl -sI https://dtizzal.com/llms.txt        # expect 200
curl -sI https://dtizzal.com/sitemap.xml     # expect 200
curl -s  https://dtizzal.com/llms.txt | tail -5   # confirm new post is at the bottom
```

If the new post isn't there, check that the workflow ran: <https://github.com/dtmirizzi/dtizzal.com/actions>.

## Performance migration script

`scripts/migrate-perf-fixes.py` is idempotent and applies four fixes across every HTML file:

- Replaces `<span class="material-symbols-outlined">NAME</span>` with inline SVG icons (saves the 318 KB icon font).
- Drops the `<link>` to `Material+Symbols+Outlined`.
- Rewrites the Courier Prime `<link>` to be non-blocking (`rel="preload"` + `onload` swap).
- Theme-toggle spans get `id="theme-icon"` preserved so `js/theme.js` can swap the SVG on click.

When to re-run:
- Hand-authored a new HTML file outside the `new-blog-post.py` workflow (the new template already includes the perf fixes).
- A merge from a branch reintroduces the icon font or blocking fonts.
- Suspect a regression: run it; if nothing changes, you're good.

```sh
python3 scripts/migrate-perf-fixes.py
```

Supported icons (the 7 used on the site): `light_mode`, `dark_mode`, `download`, `lock`, `rocket_launch`, `terminal`, `robot_2`. Adding a new icon name to a page that isn't in this set will print a warning and leave the span untouched — extend the `SVGS` dict in the script.

## Background video

`img/Background.mp4` is the looping background. Compressed to ~1 MB at 720p / CRF 24. To re-encode after changing the source:

```sh
ffmpeg -y -i source.mp4 -an -c:v libx264 -crf 24 -preset slow \
  -vf "scale=1280:720" -movflags +faststart -pix_fmt yuv420p \
  img/Background.mp4
```

Audio is intentionally stripped (the page mutes it anyway).

## Auto-generated files — don't hand-edit

- `data/blog-index.json` — regenerated by `scripts/build-blog-index.js`
- `sitemap.xml` — regenerated by `scripts/build-sitemap.js`
- `llms.txt` — regenerated by `scripts/build-llms-txt.js`

All three rebuild on every push via `.github/workflows/build-blog-index.yml`.
